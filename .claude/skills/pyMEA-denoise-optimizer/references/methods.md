# ノイズ除去手法リファレンス

`optimize_denoise.py` が試す各手法の原理・対象ノイズ・注意点。詳細な実証結果・設計判断は
リポジトリの `docs/prd/denoising.md`(PRD)を参照。

## 手法一覧

### highpass / bandpass(ゼロ位相 Butterworth)
- **原理**: Butterworth を `scipy.signal.sosfiltfilt` で順逆2回がけ(ゼロ位相)。指定帯域のみ通す。
- **対象**: ドリフト(低周波)、帯域外の広帯域ノイズ、電源(帯域設定で)。
- **長所**: ゼロ位相でピークタイミングを変えない(ISI・伝導速度を壊さない)。
- **注意**: 帯域選定が肝。心筋で上限を300Hzなど狭くすると鋭い脱分極スパイクを潰す → 心筋は上限1000〜2000Hz、神経は100-3000Hz が目安。
- **pyMEA**: `mea.data.highpass(cutoff)` / `mea.data.bandpass(low, high)`

### CMR(共通中央値リファレンス)
- **原理**: 各時刻で全64電極の中央値を減算。電極間で相関する共通モードノイズ(電源・参照ドリフト・振動)を除去。
- **長所**: 64ch の空間冗長性を活用。安価・無歪み。
- **注意**: 心筋の同期拍動では信号も一部引きうる(中央値で影響限定)。神経では効果が薄いことがある。
- **pyMEA**: `mea.data.common_median_reference()`

### wavelet 縮退
- **原理**: DWT で多重解像度分解 → ノイズ係数をソフト閾値(universal threshold)で縮退 → 再構成。
- **長所**: 鋭いスパイク形状を保ったまま帯域内白色ノイズを除去。強信号で最高S/N。
- **注意**: **微弱信号では弱スパイクごと削る**(取りこぼし)。振幅保持%で必ず確認。
- **依存**: PyWavelets。
- **pyMEA**: `mea.data.wavelet_denoise(wavelet="db4")`

## 細胞種の自動推定ロジック

最も信号が良い電極の PSD(Welch)で帯域パワーを比較:
- `power(300-3000Hz) > power(1-300Hz)` → **神経**(高域にスパイク帯)
- それ以外 → **心筋**(低〜中域優勢)

## 推奨ロジック(過剰デノイズの回避)

1. 各候補で S/N比・ノイズRMS・振幅保持・検出維持を測定。
2. **振幅保持 ≥ 80%** の手法を「スパイク形状を壊していない安全候補」とする。
3. 安全候補の中で **最大S/N** を推奨。
4. 別案として「S/N最大(振幅は犠牲)」も提示(可視化優先時用)。

検出維持%を主基準にしない理由: `detect_peak` が相対閾値(×SD)で動くため、デノイズでノイズSDが
変わると検出数が変わり、弱信号で交絡する。振幅保持の方が「スパイクを削っていないか」を直接見られる。

## 既存手法との違い

- `FilterType.FILTER_MEA`(電源同期減算)/ `iirnotch`: 電源ハム専用。広帯域ノイズに無効。
- `FilterType.CARDIO_AVE_WAVE`(拍動トリガ平均): S/Nは上がるが**時系列を1拍に潰す**(ISI等が消える)。本スキルの手法は全時系列を保持するので用途が異なる。
