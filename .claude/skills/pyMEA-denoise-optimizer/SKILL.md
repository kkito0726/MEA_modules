# pyMEA ノイズ除去 最適化スキル

MEA計測データに最適なノイズ除去条件を、**実データで探索して**提案するためのガイド。
「どのフィルタをどのパラメータで使えば一番S/Nが上がるか」を、推測ではなく測定で決める。

## このスキルがすること

1. ユーザーの `.hed` ファイルを読み込み、信号特性を解析する。
2. 周波数成分(PSD)から**細胞種(心筋/神経)を自動推定**し、ユーザーに確認・上書きさせる。
3. 細胞種に応じた候補手法(プリセット + パラメータ微調整)を適用し、S/N比・ノイズ低減・振幅保持・検出維持を比較する。
4. **最適条件を推奨**(+より強力な別案)し、pyMEA での適用方法を提示する。

出力は**推奨レポートのみ**。デノイズ適用・保存は行わない(ユーザーが推奨条件を見て自分で適用する)。

## 手順

### 1. 入力を確認する

ユーザーから `.hed` ファイルのパスを受け取る。同じディレクトリに対の `.bio` が必要。**`.bio` は `<hed名>_0001.bio` のような連番命名のことがある**(例: `foo.hed` に対し `foo_0001.bio`)。同名 `.bio` が無く見えても連番ファイルの有無を確認してから「無い」と判断する。
パスが不明・`.bio` が本当に無い場合は確認する。読み込み区間(`--start` / `--end` 秒)と電極間距離(`--distance` μm、既定450)も必要なら聞く。長すぎると重いので、まずは数秒(既定 0〜5秒)で十分。

### 2. 探索スクリプトを実行する

`scripts/optimize_denoise.py` が、読み込み→特性解析→細胞種推定→候補手法の適用→S/N比評価→推奨までを一括で行う。**リポジトリルートから** `python3` で実行する(`PYTHONPATH` にリポジトリルートを指定して pyMEA のリポジトリ版を import するため):

```bash
PYTHONPATH=<repo-root> python3 <repo-root>/.claude/skills/pyMEA-denoise-optimizer/scripts/optimize_denoise.py \
    --hed <path-to.hed> --start 0 --end 5 --distance 450
```

- インタプリタは `python3`(Python 3.12 系)を使う。リポジトリ同梱の `./venv` は壊れていることがあるので使わない。
- 細胞種を指定したい場合: `--cell-type cardio` または `--cell-type neuro`(既定 `auto`)。
- JSONも欲しい場合: `--json`。
- PyWavelets が無い環境では wavelet 系は自動でスキップされる(導入を促してよい)。

#### スクリプトを実行できない場合のフォールバック(重要)

サンドボックス・権限ポリシー等で `python3` 実行が拒否される/失敗することがある。その場合は **数値を推測で捏造しない**(このスキルは測定で決める設計)。代わりに:

1. ユーザーに、上のコマンドを**そのままコピペできる形**で提示し、自分のターミナルで実行して出力を貼ってもらうよう依頼する(`--hed` をユーザーのパスに置換)。出力を受け取ったら手順3以降(結果の解釈・細胞種確認・適用方法)を実施する。
2. 暫定の出発点として「細胞種ごとの典型的な推奨」(本ファイル末尾)を提示してよいが、**必ず「未測定の暫定値であり、上記スクリプトの実測で確定する必要がある」と明記**する。実測S/N値・振幅保持%は出さない(出せない)。

### 3. 結果を解釈してユーザーに提示する

スクリプトの出力には、信号特性・手法比較表・推奨条件(★)・別案が含まれる。これをそのまま見せるだけでなく、次を補って説明する:

- **細胞種の確認**: 「自動判定は ○○ でした。合っていますか?違えば `--cell-type` を変えて再実行します」と確認する。判定は PSD の 1-300Hz vs 300-3000Hz の優勢で決まる(心筋は低域優勢、神経は高域優勢)。
- **推奨の理由**: なぜその手法か(例: 心筋は鋭いスパイクを保ちつつ広帯域ノイズを落とす帯域 + 共通モード除去でS/Nが最大化、振幅も保持)。
- **別案の使いどころ**: 「S/N最大・振幅は犠牲」案は、検出より可視化を優先する場合のみ。

### 4. 推奨条件の適用方法を伝える

推奨は pyMEA のプリセットまたはメソッド合成で適用できる。レポートの「pyMEAでの適用」をそのまま使う:

```python
from pyMEA import read_MEA, FilterType

# プリセットで一発(read_MEA の引数)
mea = read_MEA(hed_path, start, end, distance, FilterType.CARDIO_DENOISE)

# またはメソッド合成(任意パラメータ)
denoised = mea.data.bandpass(1, 2000).common_median_reference()
```

## 評価指標の意味(ユーザーに説明する用)

| 指標 | 意味 | 見方 |
|---|---|---|
| S/N | スパイク振幅 / 静穏区間ノイズRMS | 高いほど良い |
| 改善x | raw(無処理)比のS/N倍率 | >1 で改善 |
| ノイズ | raw比の静穏区間ノイズRMS | 低いほど良い |
| 振幅保持 | raw比のスパイク振幅 | **100%付近が理想**。低いとスパイクを削っている |
| 検出% | 高閾値で確定した実スパイクの再検出率 | **弱信号では参考値**(下記注意) |

**重要な注意点**:
- **推奨は「振幅保持」を主基準にしている**。S/Nが高くても振幅を削る(=スパイク形状を壊す)手法は、特に微弱信号で取りこぼしを生むため避ける。
- **検出維持%は弱信号で交絡する**。`detect_peak` が相対閾値(×SD)で動くため、デノイズでノイズSDが下がると同じ閾値でも検出される実スパイクの数が変わる。これは手法の優劣ではなく測定の都合なので、推奨の主基準にはしない(表には参考として出す)。
- **CMR(共通中央値リファレンス)の注意**: 心筋は全面同期拍動するため、共通モード除去で信号も一部引きうる。中央値を使うことで影響は限定的だが、振幅保持% で確認する。神経では共通モード成分が少なく効果が薄いことがある。

## 細胞種ごとの典型的な推奨(目安)

実データでの探索結果に基づく出発点。最終的には必ずスクリプトの測定結果で判断する。

- **心筋(強〜中信号)**: `highpass(1) + CMR`(`FilterType.CARDIO_DENOISE`)。ドリフトと共通モードを落とし振幅無歪み。
- **微弱な心筋**: `bandpass(1, 1000〜2000) + CMR`(`FilterType.CARDIO_DENOISE_WEAK` 近傍)。広帯域ノイズを落としつつ鋭スパイクを保持。狭帯域(上限300Hz等)はスパイクを潰すので不可。
- **神経**: `bandpass(100, 3000)`(`FilterType.NEURO_DENOISE`)。スパイク帯。下限を300でなく100にすると振幅をより保てる。`+wavelet` はノイズを大きく落とすが振幅を削るので、可視化優先時の別案。

手法の原理・帯域・根拠の詳細は `references/methods.md` を参照。
