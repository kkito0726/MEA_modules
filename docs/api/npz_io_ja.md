# .npz でデータを保存・読み込みする

計測データを **1つの小さなファイル(`.npz`)にまとめて保存**し、あとから簡単に読み込むための機能です。

- 保存: `mea.save_npz("data.npz")`
- 読込: `read_MEA_npz("data.npz", electrode_distance=450)`

---

## 何が嬉しいの？(メリット)

### 1. 📦 2つのファイルが「1つ」にまとまる

計測データはもともと **`.hed` と `.bio` の2ファイル**で1セットです(`.hed` がヘッダー、`.bio` が本体)。
`.npz` にするとこれが **1ファイルにまとまり**、サンプリングレートやGAINなどの情報も中に含まれます。

```
これまで:  data.hed  +  data0001.bio   ← 2ファイルがセットで必要
.npzなら:  data.npz                    ← 1ファイルだけで完結
```

ファイルの管理・コピー・共同研究者への受け渡しがシンプルになります。

### 2. 🪶 ファイルサイズが小さくなる

元の計測ファイル(`.hed`/`.bio`)と比べて、**約 1/2** になります(既定の int16 の場合)。

| 保存形式 | 元の計測ファイル比 |
|---|---|
| **`.npz`(既定 int16)** | **約 1/2** |
| `.npz`(float32) | 約 1/1.4 |

※ 縮む割合はデータの内容によって変わります(規則的・静かなデータほどよく縮みます)。
※ pyMEA で読み込んだ後のメモリ上のデータ(float64)と比べると、int16 で約 1/7・float32 で約 1/5.5 になります。

長時間・多数の計測データを保存・持ち運ぶときに効きます。

### 3. ⚡ 読み込みが速い・簡単

`.npz` は保存済みのデータをそのまま展開するだけなので、`.hed`/`.bio` から読み直すより速く、
**コード1行**で読み込めます(`.hed`/`.bio` は不要)。

### 4. ✂️ 必要な区間だけ保存できる

長い計測のうち「ここだけ解析したい」という区間を切り出して、小さく保存できます(後述)。

---

## 使い方(3ステップ)

```python
from pyMEA import read_MEA, read_MEA_npz

# 1. いつも通り計測データを読み込む
mea = read_MEA("data.hed", start=0, end=5, electrode_distance=450)

# 2. .npz で保存する(これだけ)
mea.save_npz("data.npz")

# 3. 次回からは .npz を読み込むだけ(.hed/.bio は不要)
mea = read_MEA_npz("data.npz", electrode_distance=450)
```

読み込んだ後は、通常の `read_MEA` と全く同じように解析・描画できます。

```python
peak_index = detect_peak_neg(mea.data)
isi = mea.calculator.isi(peak_index, ch=32)
mea.fig.showAll()
```

## どちらの保存形式(dtype)を使う？

**基本は何も指定しなくてOK**です(自動で最も小さい `int16` になります)。

```python
mea.save_npz("data.npz")                     # 既定: 最小サイズ(int16)
mea.save_npz("data.npz", dtype="float32")    # 元の値を厳密に保ちたいとき
```

| 指定 | ひとことで言うと | サイズ |
|---|---|---|
| 何も指定しない (`int16`) | いちばん小さい。解析結果は変わらない | 最小 |
| `dtype="float32"` | 元の値を1ビットも変えずに保つ | 小 |

> **int16 でも解析に影響はありません。** わずかな丸めが入りますが、その誤差は
> 測定器が元々区別できる最小単位(分解能)よりさらに小さいため、ピーク検出・ISI・FPD などの
> 結果は変わりません。「元の数値を厳密に1ビットも変えたくない」場合だけ `float32` を選べばOKです。

## ✂️ 必要な区間だけ切り出して保存する

元のファイルが大きくても、解析したい区間だけを抜き出して軽い `.npz` にできます。

```python
# 大きな計測から「30秒〜40秒」だけ読み込む(全体は読み込まない)
mea = read_MEA("huge_recording.hed", start=30, end=40, electrode_distance=450)

# さらに細かく絞り込みたいとき(小数秒もOK)
mea = mea.from_slice(32.5, 35.0)

# 着目した区間だけを小さなファイルとして保存
mea.save_npz("clip_32.5-35.0s.npz")
```

「どの区間を切り出したか」もファイルに記録されるので、あとから見ても分かりやすくなります。

---

## ⚠️ 大切な注意: 計測の生データ(.hed/.bio)は消さないこと

**`.npz` は解析を楽にするための「コピー」であって、生データの代わりにはなりません。
元の計測データ(`.hed`/`.bio`)は必ずバックアップとして残し、削除しないでください。**

- `.npz`(既定の int16)は、ほんのわずかですが元の値を丸めて保存しています(=元と完全に同じではない)。
- 一度消してしまった生データは二度と戻せません。`.npz` から生データには戻せません。

➡️ **生データ = 大切な原本(必ず保管)** / **`.npz` = 作業用の軽いコピー** と考えてください。

## 参考: もう少し詳しい仕様

- **時刻データは保存していません**。読み込むときに、開始時刻・サンプリングレート・データ長から
  正確に計算し直します(`mea[0]` / `mea.times` は常に正確な時刻を返します)。
- **NumPy標準の形式**なので、pyMEA を使わなくても `np.load("data.npz")` で中身を開けます。

### .npz の中身(キー)

| キー | 内容 |
|---|---|
| `voltages` | 電位データ (64電極 × データ点数)。int16 または float32 |
| `sampling_rate` / `gain` | サンプリングレート / GAIN |
| `start` / `end` | 読み込んだ区間 (秒) |
| `dtype` / `scale` | 保存形式 / int16を元に戻すための係数 |
| `hed_path` | 元の `.hed` ファイルのパス |

### API

| 関数 / メソッド | 説明 |
|---|---|
| `PyMEA.save_npz(path, dtype="int16")` | 計測データを `.npz` で保存する |
| `read_MEA_npz(path, electrode_distance)` | `.npz` を読み込み `PyMEA` を返す(`read_MEA` と同じ使い心地) |
