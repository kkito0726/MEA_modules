# Dev Container を使った開発環境構築ガイド

初めてプログラミングに触れる方でも迷わずセットアップできるよう、ひとつひとつ丁寧に説明します。

---

## Dev Container とは？

**Dev Container（開発コンテナ）** とは、プログラムを動かす「専用の箱」をパソコンの中に作る仕組みです。

### イメージで理解する

```
あなたのパソコン
┌───────────────────────────────────────┐
│                                       │
│  ┌────────────────────────────────┐   │
│  │  Dev Container（専用の箱）     │   │
│  │                                │   │
│  │  Python 3.12                   │   │
│  │  必要なライブラリ（NumPy 等）  │   │
│  │  VS Code の拡張機能            │   │
│  │                                │   │
│  │  /workspaces/MEA_modules/data  │   │
│  │    ↑ パソコンの data/ と同期   │   │
│  └────────────────────────────────┘   │
│                                       │
└───────────────────────────────────────┘
```

この「箱」の中には、このプロジェクト（pyMEA）を動かすために必要なものがすべて入っています。

---

## なぜ Dev Container を使うの？（メリット）

### 1. 「自分のパソコンでは動いたのに…」が起きない
Python のバージョンやライブラリのバージョンが人によって違うと、同じコードが動いたり動かなかったりします。Dev Container を使うと全員が **まったく同じ環境** で作業できます。

### 2. パソコンを汚さない
Python やライブラリは「箱」の中にインストールされます。自分のパソコンに直接インストールしないので、後で不要になったら箱ごと削除するだけです。

### 3. セットアップが一瞬で終わる
「どのライブラリをインストールするか」の設定がすでに用意されています。コマンド1つで環境が自動的に整います。

### 4. チームで同じ環境を共有できる
設定ファイルがリポジトリに含まれているため、チームメンバー全員が同じ開発環境を使えます。

## 事前準備（必要なソフトウェア）

以下の4つをインストールしてください。すべて無料です。

### ステップ 1：Git をインストールする

> **Git とは？** ファイルの変更履歴を管理するツールです。リポジトリ（プロジェクトのファイル一式）をダウンロードするために使います。

- **Windows**: [Git for Windows](https://gitforwindows.org/) からダウンロード → インストーラーを起動して「Next」を押し続ける
- **Mac**: ターミナルを開いて `git --version` を実行 → インストールを促すダイアログが表示されたら指示に従う

インストール確認（Mac はターミナル、Windows は Git Bash を開いて実行）:
```bash
git --version
# git version 2.x.x と表示されれば OK
```

---

### ステップ 2：Docker Desktop をインストールする

> **Docker とは？** 「専用の箱（コンテナ）」を作るためのソフトウェアです。Dev Container はこれを使って動きます。

1. [Docker Desktop 公式サイト](https://www.docker.com/products/docker-desktop/) にアクセス
2. お使いの OS（Windows / Mac）向けのインストーラーをダウンロード
3. インストーラーを起動してインストール
4. インストール後、Docker Desktop を起動する（タスクバーやメニューバーにクジラのアイコンが表示されれば起動中）

> **注意（Windows の場合）**: WSL 2 のインストールを求められたら画面の指示に従ってインストールしてください。

インストール確認:
```bash
docker --version
# Docker version 2x.x.x と表示されれば OK
```

---

### ステップ 3：Visual Studio Code（VS Code）をインストールする

> **VS Code とは？** 無料で使えるコードエディタ（プログラムを書くためのソフト）です。Dev Container との連携機能が充実しています。

1. [VS Code 公式サイト](https://code.visualstudio.com/) からダウンロード
2. インストーラーを起動してインストール

---

### ステップ 4：Dev Containers 拡張機能をインストールする

VS Code を開いて、以下の手順で拡張機能をインストールします。

1. VS Code を起動
2. 左側のアイコンから **拡張機能（四角が4つのアイコン）** をクリック
3. 検索ボックスに `Dev Containers` と入力
4. **「Dev Containers」**（Microsoft 製）を選んで「インストール」をクリック

---

## リポジトリのクローンから Dev Container を開くまで

### ステップ 1：リポジトリをクローンする

> **クローンとは？** GitHub 上にあるプロジェクトのファイル一式を、自分のパソコンにコピーすることです。

**Mac** はターミナル、**Windows** は **Git Bash** を開き、以下のコマンドを実行します。

> **Git Bash の開き方（Windows）**: スタートメニューで「Git Bash」と検索して起動してください。コマンドプロンプトや PowerShell とは異なるので注意してください。

```bash
# ~/Workspace フォルダを作成（すでにある場合はスキップされる）
mkdir -p ~/Workspace

# Workspace フォルダに移動
cd ~/Workspace

# リポジトリをクローン
git clone https://github.com/kkito0726/MEA_modules.git
```

実行後、`~/Workspace/MEA_modules` というフォルダが作成されます。

---

### ステップ 2：計測データを配置する

> **重要**: MEA 計測データ（`.hed` / `.bio` ファイル）は Git で管理しません。専用の `data` フォルダを作り、そこにコピーします。

クローンしたフォルダの中に `data` フォルダを作成し、計測データをコピーしてください。

```bash
# data フォルダを作成
mkdir ~/Workspace/MEA_modules/data

# 計測データをコピー（例）
cp /path/to/your/measurement.hed ~/Workspace/MEA_modules/data/
cp /path/to/your/measurement.bio ~/Workspace/MEA_modules/data/
```

フォルダ構成のイメージ:

```
MEA_modules/
├── data/               ← ここに計測データを置く（Git 管理外）
│   ├── sample.hed
│   └── sample.bio
├── pyMEA/
├── .devcontainer/
└── ...
```

この `data` フォルダはコンテナ内の `/workspaces/MEA_modules/data` に自動的にバインドされます。コンテナ内からもパソコン上のファイルをそのまま読み書きできます。

コード内でのパス指定例:

```python
from pyMEA import *

# data フォルダ内のファイルを読み込む
mea = read_MEA("data/sample.hed", start=0, end=5, electrode_distance=450)
```

---

### ステップ 3：VS Code でフォルダを開く

```bash
# クローンしたフォルダを VS Code で開く
code ~/Workspace/MEA_modules
```

または、VS Code のメニューから **「ファイル」→「フォルダーを開く」** で `~/Workspace/MEA_modules` フォルダを選択します。

---

### ステップ 4：Dev Container で再度開く

VS Code でフォルダを開くと、右下に以下のような通知が表示されます。

```
┌─────────────────────────────────────────────────┐
│ フォルダーにデブコンテナー構成ファイルが含まれています。 │
│                                                   │
│  [コンテナーで再度開く]      [後で]               │
└─────────────────────────────────────────────────┘
```

**「コンテナーで再度開く」** をクリックしてください。

> 通知が表示されない場合は、VS Code 左下の **緑色のアイコン（`><`）** をクリック → 「コンテナーで再度開く」を選択します。

---

### ステップ 5：環境の自動セットアップを待つ

初回は Docker イメージのダウンロードとライブラリのインストールが自動で行われます。

```
[初回のみ] 数分かかる場合があります
  ↓
Python 3.12 の環境を準備中...
  ↓
必要なライブラリをインストール中 (NumPy, SciPy, Matplotlib 等)...
  ↓
VS Code の拡張機能をインストール中...
  ↓
完了！
```

VS Code の左下が以下のように変われば成功です:

```
Dev Container: Python 3
```

## よくあるトラブル

### Q. 「コンテナーで再度開く」の通知が表示されない
**A.** VS Code 左下の緑色の `><` アイコンをクリックして「コンテナーで再度開く」を選択してください。

### Q. Docker Desktop が起動していないと言われる
**A.** Docker Desktop アプリを起動してから（タスクバー / メニューバーにクジラアイコンが表示されるまで待つ）、再度試してください。

### Q. 「WSL 2 インストールが必要」と表示される（Windows）
**A.** 表示されたリンクから WSL 2 をインストールしてください。インストール後にパソコンの再起動が必要な場合があります。

### Q. コンテナのビルドが途中で止まる
**A.** インターネット接続を確認してください。ライブラリのダウンロード中はネット接続が必要です。
